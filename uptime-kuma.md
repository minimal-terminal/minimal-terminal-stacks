# Uptime Kuma\n\nUptime Kuma is a free, open-source, and self-hosted monitoring tool. It's a fancy, open-source alternative to \"Uptime Robot\" that supports multiple notification channels, providing beautiful status pages and comprehensive service health checks.\n\n## Prerequisites\n\n*   A running Docker environment (e.g., on a home server or Synology NAS).\n*   Ensure port `3001` is available on your host system.\n*   A dedicated Docker volume for persistent data is recommended to retain your monitoring configurations and history across container restarts.\n\n## Docker Compose Script\n\n```yaml\nversion: '3.8'\nservices:\n  uptime-kuma:\n    image: louislam/uptime-kuma:1\n    container_name: uptime-kuma\n    ports:\n      - \"3001:3001\" # Host_Port:Container_Port\n    volumes:\n      - uptime-kuma-data:/app/data\n    restart: unless-stopped\n    environment:\n      # Optional: Set your desired timezone, e.g., Europe/Berlin, America/New_York\n      # Find valid timezones here: https://en.wikipedia.org/wiki/List_of_tz_database_time_zones\n      - TZ=Etc/UTC\n\nvolumes:\n  uptime-kuma-data:\n```\n\nTo deploy, save the above content as `docker-compose.yml` and run `docker compose up -d` in the same directory.
