# Uptime Kuma\n\nUptime Kuma is a fancy self-hosted monitoring tool like "Uptime Robot". It monitors HTTP(s) / TCP / HTTP(s) Keyword / Ping / DNS / Push / Steam Game Server / Docker Containers. Notifications are supported via various services.\n\n## Prerequisites\n\nBefore deploying Uptime Kuma, ensure you have:\n\n*   A Docker-enabled host (e.g., Ubuntu server, Synology NAS with Docker).\n*   A persistent volume path on your host for Uptime Kuma's data (e.g., `/path/to/your/uptime-kuma/data`).\n*   An available port on your host, typically `3001`, to access the web interface.\n\n## Docker Compose Script\n\n```yaml\nversion: '3.3'\nservices:\n  uptime-kuma:\n    image: louislam/uptime-kuma:1\n    container_name: uptime-kuma\n    volumes:\n      - /path/to/your/uptime-kuma/data:/app/data\n    ports:\n      - "3001:3001"\n    restart: unless-stopped\n    environment:\n      # Optional: Adjust timezone if needed. Example: - TZ=Europe/London\n      # - TZ=America/New_York\n      # Set PUID and PGID to match your host user for proper file permissions.\n      # You can find these by running 'id -u' and 'id -g' in your terminal.\n      - PUID=1000 \n      - PGID=1000 \n```
