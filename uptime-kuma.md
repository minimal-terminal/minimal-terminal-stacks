# Uptime Kuma\n\nUptime Kuma is a free and open-source monitoring tool that allows you to monitor your services and receive notifications when they go down. It supports HTTP(s), TCP, Ping, DNS, Push, Steam Game Server, and more, all presented through a beautiful and intuitive web interface.\n\n## Prerequisites\n\nBefore deploying Uptime Kuma, ensure you have:\n\n*   Docker and Docker Compose installed on your host system.\n*   A dedicated directory for persistent application data (e.g., `/your/appdata/path/uptime-kuma`).\n*   An available port on your host (e.g., `3001`) to map to Uptime Kuma's internal port.\n\n## Docker Compose Script\n\nBelow is a production-ready `docker-compose.yml` script for Uptime Kuma. Remember to replace `/your/appdata/path/uptime-kuma` with your desired host path for persistent data storage.\n\n```yaml\nversion: '3.8'\n\nservices:\n  uptime-kuma:\n    image: louislam/uptime-kuma:1\n    container_name: uptime-kuma\n    ports:\n      - "3001:3001" # Host_Port:Container_Port - Change host port if 3001 is in use\n    volumes:\n      - /your/appdata/path/uptime-kuma:/app/data # Persistent data storage\n    restart: unless-stopped\n\n```
