# Uptime Kuma\n\nUptime Kuma is a fancy open-source self-hosted monitoring tool like "Uptime Robot". It allows you to monitor HTTP(s), TCP, Ping, DNS, Push, Steam Game Server, and more. It features a beautiful, responsive UI and sends notifications via a wide array of services including Telegram, Discord, Slack, Email, and custom webhooks.\n\n## Prerequisites\n\n*   A server running Docker and Docker Compose.\n*   A reverse proxy (like Nginx Proxy Manager, Caddy, or Traefik) if you want to access Uptime Kuma via a custom domain with SSL.\n*   Make sure to replace `/path/to/your/uptime-kuma/data` with an actual path on your host system.\n\n## Docker Compose Script\n\n```yaml\nversion: '3.8'\n\nservices:\n  uptime-kuma:\n    image: louislam/uptime-kuma:1\n    container_name: uptime-kuma\n    volumes:\n      - /path/to/your/uptime-kuma/data:/app/data\n    ports:\n      - "3001:3001" # Host_Port:Container_Port - Change 3001 to an available port on your host if needed\n    restart: unless-stopped\n    networks:\n      - default # Or your custom Docker network if using a reverse proxy\n\nnetworks:\n  default:\n    driver: bridge\n```
